# 直流电场感应的非侵入式出口硬压板状态检测 — 算法分析与设计文档

> 日期：2026-06-15
> 基于：论文《直流电场感应的非侵入式出口硬压板状态检测》、RTU报文样例、Modbus点表、数据库建表SQL

---

## 一、项目目标

从 **达梦DM8数据库** 读取RTU采集的传感器报文数据，实现 **出口硬压板投/退状态** 的自动判别算法，最终输出每个压板 **"投"（已接通）** 或 **"退"（已断开）** 的状态。

---

## 二、核心原理回顾

### 2.1 物理原理

| 压板状态 | 下端子电压 | 传感器检测到的电场 |
|---------|-----------|------------------|
| **退（断开）** | 0 V | 弱电场 |
| **投（接通）** | -110 V DC | 强电场 |

### 2.2 传感器工作原理

采用 **场磨式直流电场传感器**（field mill）：

- 一个旋转的接地屏蔽片，周期性暴露/屏蔽感应电极
- 感应片表面电荷周期性变化 → 产生 **交流感应电流**
- 电流幅值与电场强度成 **线性关系**

论文公式：
```
i(t) = d(∑q)/dt = ε₀ · E · dA(t)/dt
```
其中 A(t) 是感应片暴露于被测电场中的总面积，E 为被测电场强度。

### 2.3 状态判别逻辑

传感器输出信号的 **幅值大小** 直接对应压板状态：

```
幅值 ≥ 阈值 → 投（压板接通，-110V存在）
幅值 < 阈值 → 退（压板断开，0V）
```

论文实验结论：区分程度 **大于3倍**，可以很好地分辨。

## 三、数据源分析

### 3.1 数据库结构（达梦DM8）

**表1：`rtu_record`** — 报文主记录
- `RTU_REGS_NODE_P1_0` ~ `P5_9`：50个传感器节点的**状态值**（0/1 或 0~255）
- `RTU_REGS_TEMP_P1_0` ~ `P1_9`：P1通道10个传感器的**温度**（0.1°C）
- `RTU_REGS_HUM_P1_0` ~ `P1_9`：P1通道10个传感器的**湿度**（0.1%）
- `RTU_REGS_P00_ROTOR_RPM`：场磨传感器**转子转速**（0.1 RPM）
- `RTU_REGS_P00_ENV_TEMP` / `_ENV_HUMIDITY`：**环境温湿度**
- `RTU_REGS_P00_WAVE_BUSY`：**波形采集忙标志**（1=正在采集，0=空闲）
- `SYSTEM_TIME`：设备系统时间

**表2：`rtu_wave_data`** — P00波形数据
- `RECORD_ID`：关联 `rtu_record.ID`
- `WAVE_DATA`：**512个逗号分隔的ADC采样值**（核心数据）

### 3.3 NODE 状态字段含义

从 `slave.h` 定义看：
- `RTU_REGS_NODE_P1_0` ~ `P5_9` 各占 **1个寄存器（16位）**，标记为 `1b`
- 样本中所有 NODE 均为 0

**可能的含义**（需要确认）：
- `0` = 退（断开）
- `1` = 投（接通）
- 或者存储的是 ADC 量化后的电场幅值编码

```

│  rtu_record（主记录 + 状态 + 温湿度 + 参数）                      │
│  rtu_wave_data（512点波形数据）                                   │

                           ↓

│  ① 按 SLAVE_ID + SYSTEM_TIME 关联两张表                          │
│  ② 解析 WAVE_DATA 字符串 → float/int数组                         │
│  ③ 去除无效尾部零（如有）                                         │
│  ④ 温湿度数据换算（÷10）                                          │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│                  信号处理层（核心算法）                              │

│  ⑦ 幅值提取（RMS / FFT / 峰峰值检波）                              │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│                  状态判别层                                        │
│  ⑧ 幅值 → 标定转换（线性映射到电场强度或电压值）                       │
│  ⑨ 阈值比较 → 投/退 状态                                          │
│  ⑩ 历史趋势分析 → 检测"假投入"异常                                  │
└──────────────────────────┬──────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│                  输出层                                           │
│  输出：每个传感器节点的状态（投/退）、置信度、时间戳                    │
│  异常告警：假投入、传感器故障、通信异常                               │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 信号处理算法详解

#### 方案A：RMS检波（推荐，计算量小）

```python
1. 去直流：x'[n] = x[n] - mean(x)
2. 带通滤波：y[n] = bandpass(x'[n], fc=213.8Hz, bw=50Hz)
3. RMS计算：RMS = sqrt(mean(y[n]²))
4. 状态判断：RMS > threshold ? "投" : "退"
```

**优点**：计算简单，物理意义明确（RMS与电场强度成正比）

#### 方案B：FFT幅值提取（精度高）

```python
1. 去直流：x'[n] = x[n] - mean(x)
2. FFT：X[k] = fft(x'[n])，N=512
3. 找基频峰值：k0 = round(213.8 * N / fs) 附近的谱线
4. 幅值：A = 2 * |X[k0]| / N
5. 状态判断：A > threshold ? "投" : "退"
```

**优点**：抗干扰强，可同时监测谐波（诊断传感器故障）
**缺点**：需要知道采样率 fs

#### 方案C：峰峰值检波（最简单）

```python
1. 带通滤波后找极值
2. Vpp = max(y[n]) - min(y[n])
3. 状态判断：Vpp > threshold ? "投" : "退"
```

**优点**：极简实现
**缺点**：对噪声敏感

### 4.3 采样率的确定

需要确认采样率 fs，可以从以下途径推算：

1. **硬件文档**：查看采集器Modbus点表PDF或硬件手册
2. **已知信息**：
   - 512个采样点
   - 转子频率 ≈ 213.8 Hz
   - 一个周期内的采样点数可以推算

按奈奎斯特定理，fs ≥ 2 × 213.8 ≈ **428 Hz**。一般会留余量，可能在 **1 kHz ~ 5 kHz** 范围。

### 4.4 状态判别的关键阈值

论文指出：
- **退**态：电压 0V，传感器输出接近零
- **投**态：电压 -110V DC，传感器输出幅值大
- 区分度 > **3倍**

阈值设定策略：
1. **固定阈值**：根据标定实验确定一个统一阈值
2. **自适应阈值**：根据温湿度补偿，动态调整阈值（因为有温度传感器数据）
3. **双阈值迟滞**：防止临界区抖动

### 4.5 假投入检测逻辑

"假投入" = 压板物理上看似投入，但回路实际断开。

**检测方法**：
1. 对比 `RTU_REGS_NODE_Px_y`（位置传感器/双联压板信号）与 **算法判别结果**
2. 如果位置信号 = 投入，但电场信号 = 退出 → **疑似假投入**
3. 连续N次确认后发出告警

```
if (node_status == 1 && electric_field_status == 0) {
    false_input_counter++;
    if (false_input_counter >= CONFIRM_COUNT) {
        alarm("假投入检测: 通道Px_y");
    }
} else {
    false_input_counter = 0;
}
```

4. **数据频率**：RTU上报间隔是多少秒？用于设计实时处理的轮询周期
5. **波形数据对应关系**：一份 WAVE_DATA 对应哪个通道/P00本身是一个环境参考传感器，还是每个通道都有自己的波形数据？
