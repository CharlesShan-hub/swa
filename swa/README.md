# swa: Sine Wave Analysis

非侵入式出口硬压板状态检测 — 基于场磨式直流电场传感器波形分析。

## 原理

场磨式传感器输出正弦波信号，幅值与电场强度成正比：

| 压板状态 | 下端子电压 | 传感器输出 |
|---------|-----------|-----------|
| 退（断开） | 0 V | 幅值小（≈0） |
| 投（接通） | -110 V DC | 幅值大 |

对 512 点波形做 FFT 提取基波幅值，结合温湿度等环境参数，通过模型预测电压，与阈值比较判别。

## 输入参数一览

### 传感器硬件参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 采样率 | 15873 Hz | 场磨传感器 ADC 采样频率 |
| 每帧点数 | 512 点 | 单次采集的波形长度 |
| 转子频率 | 213.8 Hz | 场磨旋转叶片调制频率 |
| 测量范围 | -110V ~ +110V | 直流电压 |
| 数据类型 | 电压值 (V) | 已标定，非 ADC 原始值 |

### 模型输入特征（共 18 维）

#### 特征列表

| # | 特征 | 来源 | 物理含义 | 与电压相关性 |
|---|------|------|---------|-------------|
| 1~10 | A1 ~ A10 | 512 点波形 → FFT 前 10 次谐波幅值 | 各次谐波分量大小，A1（基波）为转子频率 213.8Hz 处的幅值，与电场强度成正比 | **A1 为主信号**，A2~A10 反映传感器机械状态 |
| 11 | 温度 T | 传感器内置温度计 | 环境温度，影响放大器增益和 ADC 参考电压 | 弱相关（温度补偿） |
| 12 | 湿度 RH | 传感器内置湿度计 | 环境湿度，影响绝缘电阻和漏电流 | 弱相关（湿度补偿） |
| 13 | RPM | 转子转速测量 | 场磨电机转速，影响感应电流幅值 | 中等相关（转速补偿） |
| 14 | Vpp | 波形峰峰值 Vpp = max(ac) - min(ac) | 信号最大波动范围，反映总能量 | 与 A1 相关系数仅 **0.18**，携带互补信息 |
| 15 | RMS | 波形有效值 RMS = sqrt(mean(ac²)) | 信号的有效能量，与 Vpp 相关但反映不同维度 | 与 Vpp 相关 **0.89**，但反映的是能量而非峰值 |
| 16 | CrestFactor | 波峰因子 = 峰值 / RMS | 波形"尖"的程度，标准正弦波=1.414，**实际值 1.68~3.24**，偏离越多说明畸变越大 | **有效区分畸变波与理想波** |
| 17 | Kurtosis | 峭度（Fisher =False） | 波形尖锐度，正态分布=3，**实际值 1.56~3.09**，>3 表示有尖峰 | 反映传感器异常或噪声水平 |
| 18 | Skewness | 偏度 | 波形不对称度，正=正半周更大，**实际值 -0.43~0.54** | 反映机械偏心或叶片磨损 |

#### 特征推导流程

```
512 点原始波形 (512,)
    │
    ├── 去直流：ac = wave - mean(wave)
    │
    ├── FFT ─── 前 10 次谐波幅值 ─── A1 ~ A10（频域特征）
    │
    ├── 时域统计 ─── Vpp, RMS, CrestFactor, Kurtosis, Skewness
    │
    └── 传感器 ─── T（温度）, RH（湿度）, RPM（转速）
```

#### 特征使用说明

- **线性模型**（linear_basic / linear_with_env / linear_full）：只用 A1 + 环境参数，共 4~5 维
- **二阶工程模型**（quadratic_model）：共 13 维（A1~A10 + T + RH + RPM）
- **XGBoost / LightGBM**：使用全部 13 维（A1~A10 + T + RH + RPM）
- **LeNet-Hybrid**：使用三路独立输入 — 原始波形(512点) + FFT谐波(10维) + 环境/时域参数(8维)

#### 特征相关性

| 特征对 | 相关系数 | 结论 |
|-------|---------|------|
| A1 — Vpp | **0.18** | 弱相关，**各自携带独立信息** |
| Vpp — RMS | **0.89** | 强相关但非冗余（峰值 vs 能量） |
| A1 — RMS | **0.13** | 弱相关，互补 |
| Crest — Kurtosis | **0.68** | 中等相关，共同反映波形畸变 |
| Skewness — 其他 | < 0.15 | 几乎独立，反映不对称性 |

> **备注**：XGBoost MAE 2.95V / LightGBM MAE 2.09V 是在完整 38000 条数据（含 7723 条 30V 中等电压）下的结果。30V 电压信号弱、预测难度大，导致整体 MAE 相比无 30V 数据的旧版（1.51V）有所上升。**这不是模型退步，而是评估更全面真实。** 详见下方数据分布分析。

#### 为什么 Boost 模型指标比之前下降？

| 版本 | 数据量 | 数据组成 | MAE | 说明 |
|------|--------|---------|-----|------|
| 旧版 | 30000 条 | 无 30V 数据 | **1.51 V** | 测试集不含 30V，评估不完整 |
| 新版 | 38000 条 | **含 7723 条 30V（20%）** | **2.09~2.95 V** | **评估更全面，反映真实场景** |

**核心原因**：
1. **新增 30V 数据占 20%**：30V 是中等电压，A1（基波幅值）小，信噪比低，模型难以精确预测
2. **旧版测试集不完整**：之前的 1.51V 是在没有 30V 的测试集上测的，MAE 偏低
3. **新版测试集包含真实场景**：30V、50V、80V 等各类电压都涵盖，MAE 更准确反映实际部署效果
4. **关键是 LightGBM MAE 2.09V 仍满足工程需求**：投/退判断阈值 30V，2.09V 的误差完全在可接受范围内

## 算法对比（38000 条数据，34200 训练 / 3800 测试）

### 总览

| # | 算法 | 特点 | MAE | RMSE | 最大误差 | 模型大小 | 训练耗时 |
|---|------|------|-----|------|---------|---------|---------|
| 1 | `linear_basic` | 仅基波 A1，单变量线性回归 | 44.60 V | 48.83 V | 118.49 V | ~100 B | <1 秒 |
| 2 | `linear_with_env` | A1 + 温度 + 湿度 | 44.50 V | 47.94 V | 133.28 V | ~100 B | <1 秒 |
| 3 | `linear_full` | A1 + 温度 + 湿度 + RPM | 44.50 V | 47.93 V | 132.68 V | ~100 B | <1 秒 |
| 4 | `quadratic_model` | 二阶工程模型，含 A1²、T² 等交叉项（13 参数） | 27.99 V | 36.49 V | 151.04 V | ~100 B | <1 秒 |
| 5 | `xgboost_model` | XGBoost 回归，样本加权（缓解数据不均），300 棵树 | 2.95 V | 6.96 V | 93.03 V | ~10 KB | ~5 秒 |
| 6 | `lightgbm_model` | LightGBM 回归，速度与精度均衡 | **2.09 V** | **6.14 V** | **76.68 V** | ~10 KB | ~5 秒 |
| 7 | `lenet` | LeNet-1D 卷积，**直接吃原始 512 点波形**，17K 参数 | 8.12 V | 8.38 V | 16.42 V | ~200 KB | ~10 分钟 |
| 8 | `lenet_hybrid` | **LeNet-Hybrid（精度王）**，三路融合 + 时域特征，37K 参数 | **0.99 V** | **3.05 V** | **79.09 V** | ~500 KB | ~15 分钟 |

结论：
- **传统线性模型（1~3）**：误差 44V，仅基波幅值不足以预测电压，温湿度 RPM 几乎没有帮助 — 原因是传感器在不同电压下输出信号的谐波结构复杂，线性模型无法拟合。
- **二阶工程模型（4）**：MAE 27.99V，加入非线性项后有所改善，但仍远不达标 — 说明电压与特征之间存在更高阶的非线性关系。
- **XGBoost / LightGBM（5~6）**：MAE 2~3V，树模型能自动捕捉复杂非线性关系，训练快、部署轻量。**LightGBM 为生产轻量首选**。
- **LeNet-1D（7）**：MAE 8.12V，直接从原始波形学习，无需 FFT 特征提取，适合嵌入式端到端部署。
- **LeNet-Hybrid（8）**：**MAE 0.99V，全量数据最佳结果**。三路输入（时域 Conv1D + 频域 FFT 特征 + 环境/时域参数）融合，配合 Xavier 初始化 + Mini-Batch + 学习率衰减，参数/数据比 1.2:1。

### LeNet-Hybrid 演进史

LeNet-Hybrid 从最初到最佳经历了多轮迭代，关键技术改进：

| 版本 | 参数量 | 参数/数据比 | 特征 | 关键改进 | MAE | RMSE | 最大误差 |
|------|--------|-----------|------|---------|-----|------|---------|
| v0（原始） | 273 K | 11.4:1 | A1~A10 + T/RH/RPM | Full Batch，无归一化，无早停 | 2.61 V | 3.19 V | 13.12 V |
| v1（归一化） | 273 K | 11.4:1 | + 环境参数 z-score 归一化 | 特征归一化，但**过拟合严重** | 1.66 V | 2.08 V | 9.40 V |
| v2（缩减 17K） | 17 K | 0.73:1 | + 网络缩小（Pool 4×4） | 参数量大幅降低，但**欠拟合** | 3.00 V | 4.79 V | 47.46 V |
| v3（30K 条 26K） | 26 K | 1.09:1 | + 网络适中调整 | **参数比接近 1:1，不过拟合** | 2.13 V | 4.08 V | 57.75 V |
| v4（Mini-Batch） | 26 K | 1.09:1 | + Xavier 初始化 + DataLoader | 训练更稳定，加速收敛 | 2.13 V | 4.08 V | 57.75 V |
| v5（38K 条 36K） | 36 K | 1.20:1 | + 数据量增至 38000 条 + 网络扩大 | **数据增多 + 容量匹配，爆发式提升** | 1.21 V | 3.20 V | 69.38 V |
| **v6（时域特征，8维）** | 37 K | 1.21:1 | + Vpp + RMS + Crest + Kurtosis + Skewness | 与 FFT 特征低相关（r<0.2），提供互补信息 | **0.99 V** | **3.05 V** | **79.09 V** |
| v7（时域缩减，6维） | 37 K | 1.21:1 | -RMS -Crest | 去掉 RMS 和 Crest 后精度暴跌（2.24V），证实有效 | 2.24 V | 3.76 V | 62.27 V |
| **v8（谐波优化 12）** | **37 K** | **1.21:1** | **A1~A12（PCA 显示 12 维覆盖 99% 方差）** | **固定 100 轮对比实验，MAE 4.83V，比 A1~A10 好 35%** | **待跑** | | |

### 各算法适用场景

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| **生产部署（CPU、轻量）** | `lightgbm_model` | MAE 2.09V，~5 秒训练，~10 KB 模型，纯 CPU |
| **生产部署（CPU、需要权重公平）** | `xgboost_model` | 样本加权，对 -40V（占 55%）降权，对罕见电压加权 |
| **端到端（无需 FFT 提取）** | `lenet` | 原始波形直出，适合嵌入式设备 |
| **最高精度（有 GPU）** | `lenet_hybrid` | MAE **0.99V**，三路融合 + 时域特征，参数比 1.2:1 |
| **快速验证、传统方案** | `quadratic_model` | 无需训练库，纯数学公式 |

### 运行

数据源由 `src/swa/config/settings.py` 中的 `data_source.mode` 控制：

```python
data_source.mode = "db"     # 从达梦数据库读取（在公司用）
data_source.mode = "local"  # 从本地 JSONL 文件读取（回家离线用）
```

```bash
# 按配置模式运行（默认最近20条）
uv run python main.py

# 指定条数
uv run python main.py --limit 100

# 按从机地址和时间筛选
uv run python main.py --slave-id 1 --since "2026-06-01"

# 完整分析 + 生成 2×2 图表
uv run python main.py --plot
```

### 导出数据到本地（回家离线用）

在公司执行，分批导出不伤数据库：

```bash
uv run python scripts/export_data.py --limit 10000 --batch 200 --sleep 2
```

然后回家把 `settings.py` 里的 `mode` 改为 `"local"` 即可。

## 算法对比（30000 条数据，25000 训练 / 5000 测试）

| # | 算法 | 公式 | MAE | RMSE | 最大误差 |
|---|------|------|-----|------|---------|
| 1 | `linear_basic` | V = b0 + b1·A1 | 36.74 V | 36.88 V | 44.12 V |
| 2 | `linear_with_env` | + b2·T + b3·RH | 26.90 V | 27.48 V | 44.22 V |
| 3 | `linear_full` | + b4·RPM | 26.90 V | 27.49 V | 43.88 V |
| 4 | `quadratic_model` | 二阶工程模型（13 参数） | **4.71 V** | **6.88 V** | **28.05 V** |
| 5 | `xgboost_model` | XGBoost 回归（含正则化） | **1.51 V** | **5.99 V** | 57.41 V |
| 6 | `lenet` | LeNet-1D 卷积（原始波形，17K 参数） | 8.12 V | 8.38 V | 16.42 V |
| 7 | `lenet_hybrid` | LeNet-Hybrid（时域+频域+环境，26K 参数） | **2.13 V** | **4.08 V** | 57.75 V |

结论：
- 传统线性模型（1~3）误差大，无法满足工程需求
- **二阶工程模型**（4）MAE 4.71V，可用于粗略估计
- **XGBoost**（5）MAE 最低（1.51V），训练快、轻量、无需 GPU，**性价比最高，推荐生产首选**
- **LeNet-Hybrid**（7）MAE 2.13V，参数/数据比 1.09:1，完全无过拟合，适合深度学习研究场景

> 注：XGBoost 启用 L1/L2 正则化后，RMSE 和最大误差均优于无正则化版本，模型更稳定。

## 训练与预测

### 训练模型

```bash
uv run python scripts/train_model.py --algorithm linear_basic
uv run python scripts/train_model.py --algorithm quadratic_model
uv run python scripts/train_model.py --algorithm xgboost_model
uv run python scripts/train_model.py --algorithm lightgbm_model
```

不指定 `--algorithm` 时，默认使用 `settings.py` 中配置的算法。

### 预测与状态判别

```bash
uv run python main.py --limit 20
```

输出格式：
```
  110V  slave=1  2026-06-10 10:58:39  电压= +99.94V  投
   -50V  slave=1  2026-06-10 17:24:16  电压= -48.23V  投
    10V  slave=1  2026-06-11 09:15:33  电压=  +8.12V  退
```

### 投/退判断逻辑

```
预测电压的绝对值 > threshold_abs → "投"（有电场，压板接通）
预测电压的绝对值 < threshold_abs → "退"（无电场，压板断开）
```

阈值 `threshold_abs` 在 `settings.py` 中设置，默认 30V。
